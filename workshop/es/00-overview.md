# 🎮 VS Code GitHub Copilot Agent Lab

[📚 Guía del Lab](https://copilot-dev-days.github.io/agent-lab-java/docs/) • [← README](../README.md)

---

> **Duración:** ~1 hora
> **Nivel:** Intermedio
> **Stack:** Java 21 / Spring Boot 3.4.2 / Apache Maven

Bienvenido a Soc Ops — un workshop práctico donde transformarás una aplicación simple de Social Bingo en algo increíble usando Agent Mode de VS Code con GitHub Copilot.

---

## 📋 Lista Rápida

Antes de comenzar, verifica:

- [ ] VS Code **v1.107+** (sin actualizaciones pendientes)
- [ ] Sesión iniciada con **GitHub Copilot** (Pro, Business o Enterprise)
- [ ] Git instalado
- [ ] Java 21 JDK instalado
- [ ] Panel de Chat abierto y Agent listo

> 💡 **Consejo:** ¡Usa el DevContainer para un entorno preconfigurado!

---

## 🎯 Lo que Aprenderás

| # | Habilidad | Descripción |
|---|-----------|-------------|
| 1 | **Ingeniería de Contexto** | Enseñarle a la IA sobre tu código con instrucciones |
| 2 | **Primitivas Agénticas** | Usar agentes en segundo plano, agentes en la nube y flujos personalizados |
| 3 | **Desarrollo Orientado al Diseño** | Dejar que la IA itere en la UI mientras tú guías la visión |
| 4 | **Desarrollo Guiado por Pruebas** | Usar agentes TDD para un desarrollo confiable de funcionalidades |

---

## 📚 Partes del Lab

| Parte | Título | Tiempo | Descripción |
|-------|--------|--------|-------------|
| [**01**](01-setup.md) | Configuración e Ingeniería de Contexto | 15 min | Clona, configura y enséñale a la IA sobre tu proyecto |
| [**02**](02-design.md) | Desarrollo Frontend Orientado al Diseño | 15 min | Rediseña la UI con temas creativos |
| [**03**](03-quiz-master.md) | Quiz Master Personalizado | 10 min | Crea tus propios temas de quiz con agentes personalizados |
| [**04**](04-multi-agent.md) | Desarrollo Multi-Agente | 20 min | Construye nuevas funcionalidades con TDD y agentes de diseño |

---

## 💡 Consejos

1. **Mantén el navegador abierto** — Observa las actualizaciones en tiempo real mientras programas
2. **Haz commit con frecuencia** — Guarda los estados funcionales regularmente
3. **Usa checkpoints** — Revierte cambios inesperados con Checkpoints y Undo del chat
4. **📌 Fija esta guía** — Mantenla visible mientras trabajas
